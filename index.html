4.2 批量下单

基本信息

Path： /api/v1/cash/orders

Method： POST

接口描述：

功能：现货交易对 - 批量委托下单 （下多笔委托订单）
注意：
  交易对ID：contractId，交易对名称：symbol 下单时，必须至少传递一个，当两个都传递时，用contractId
  客户端委托ID：clientOrderId，不传递时，内部会自动生成
请求响应数据-data格式如下所示：
    "data":{"succ":[[${客户端委托ID]],"reject":[[${客户端委托ID},${失败码}]]}

请求参数

Headers

参数名称 参数值 是否必须 示例 备注
Content-Type application/json 是
signature 是 签名
apiKey 是 公钥
apiExpires 是 时间戳

Body

名称类型是否必须默认值备注其他信息
ordersobject []非必须

item 类型: object

├─ contractIdinteger非必须交易对ID
├─ symbolstring非必须交易对名称
├─ sideinteger必须买1，卖-1
├─ orderPricestring非必须委托价格
├─ orderQtystring必须委托数量
├─ orderTypenumber必须1（限价），3（市价）
├─ timeInForceinteger必须1（取消前有效）
├─ clientOrderIdstring非必须客户端委托ID

返回数据

名称类型是否必须默认值备注其他信息
codenumber非必须0:成功，非0:失败
msgstring非必须消息
dataobject非必须
├─ succarray []非必须下单成功列表

item 类型: array

├─ rejectstring []非必须下单失败列表

item 类型: string
